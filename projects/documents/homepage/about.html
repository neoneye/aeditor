What is this project about?

Pair-programming can be unnecessary confusing, without a good editor. Its difficult to keep an overview when your not sitting at the keyboard. When hitting pageup/down the observer quickly looses his bearings. What is needed is animation, scrolling, folding and assistence with refactoring.

Please take your time, and write a message in my guestbook, >> click here. Or if you dare then visit my homepage.

Implementation plan

  • use FreeBSD license.
  • write backend in Ruby to obtain flexibility and robustness.
  • write widget-frontends in C++ to obtain speed.

Experimental goals

  • experiments with animated operations. Will this reduce the observers confusion?
  • dual-screen experiments. How does it work out if one is writing code and the other one is writing comments? What if you can hand over the keyboard to the observer, just by pressing F12, so that you become the observer?
  • improve user experience.. HCI + contextual design.
