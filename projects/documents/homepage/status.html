Iteration 3 - Usable

Many editing operations is working. Still regex is missing. Goal for me is to glue the editor-backend (Ruby) together with the frontends (C++).

The AEditor-backend is written in Ruby, where I use test-first development. Status at this very moment is: 403 tests, 1407 assertions, 0 failures, 0 errors. I have never used TDD in such big scale, it really rocks. Lack of unit-testing was a major problem with earlier iterations of this project, things were hard(impossible) to debug. There is a screenshot of the Ncurses frontend [here].

The Fox-AEditor-frontend is written in C++, and is at a very early stage. I hope to embed this widget into FreeRIDE some day.

I am conducting some experiments with embedding [ruby into c++]. Which I use for the GUI-widgets (Fox, GTK, QT, OpenGL).

The current top-entries in my todo-list says; Implement markers. Make the Fox-widget usable. See more status [TODO, FEATURES, CHANGES].

Iteration 2 - Prototype

The primary goal was to make editing possible, which I achieved. It has a very flexible editing engine which supports folding, wordwrap, symbols, markers. It has a seperate ncurses frontend. The overall design is clean. Unfortunatly did blocking not fit into this design, which send me back to the drawing board. Thus iteration 2 is paused! Iteration 3 has become so good that Iteration 2 is completely dead.

Iteration 1 - Prototype

This only worked with BorlandC31 (MSDOS only). It was capable of: line-folding, scope-folding, wordwrap, hiliting, undo/redo. I had unfortunatly underestimated the need for editing, so it has only very limited editing capabilities. Its design was flawed, masochistic, inflexible and non-scalable.
